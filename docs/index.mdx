Yieldo is a cross-chain deposit aggregator that lets users deposit into yield vaults from any supported chain and token. The API handles routing, bridging, and vault deposits with no fees to the user. An on-chain attribution system tracks deposits for revenue share with vault partners.

## Base URL

```
https://api.yieldo.xyz
```

## How It Works

1. **Browse vaults** - Query available vaults across chains and protocols
2. **Get a quote** - Submit your source chain, token, and amount to get a deposit quote with route options
3. **Build the transaction** - Submit the quote data to get a ready-to-send transaction
4. **Send & track** - Send the transaction and monitor the deposit progress

## Key Concepts

### Attribution-Only Router

Yieldo's deposit router is non-custodial — it pulls tokens from the user, deposits them into the vault, and emits a `Routed` event for on-chain attribution. No fees are deducted. 100% of the user's deposit goes into the vault.

Each deposit carries a `partnerId` and `partnerType` for attribution — identifying which wallet, KOL, or integration referred the user. This is used for revenue share tracking with vault partners.

### Cross-Chain Routing

For cross-chain deposits, Yieldo uses [LiFi](https://li.fi/) to bridge and swap tokens. Users can choose from multiple bridge routes (like Jumper.exchange), with the best route auto-selected.

### Vault Adapter Pattern

The router supports any vault type through a pluggable adapter system. Standard ERC-4626 vaults work out of the box. Custom vaults (Midas, Veda, Lido, etc.) use adapters that can be deployed without upgrading the router contract.

| ----------- | -------- | ----------------- |

| Optimism    | 10       | Source + Vaults    |
| Monad       | 143      | Source + Vaults    |

| Avalanche   | 43114    | Source only        |
| BSC         | 56       | Source only        |

### Supported Vault Types

| Type     | Description                          | Examples                                      |
| -------- | ------------------------------------ | --------------------------------------------- |
| ERC-4626 | Standard tokenized vault interface   | Steakhouse USDC, Gauntlet WETH Prime, Moonwell Flagship USDC |
| Custom   | Protocol-specific via adapters       | Midas mTBILL, Veda Liquid ETH, Lido Earn, IPOR |

### Revenue Model

Yieldo charges **no fees to users**. Revenue comes from revenue share agreements with vault curators and protocols. Every deposit routed through Yieldo is attributed on-chain via the `Routed` event, providing provable attribution for invoicing.

## Quick Example

```bash
# 1. List vaults on Base
curl https://api.yieldo.xyz/v1/vaults?chain_id=8453

# 2. Get a quote to deposit 1000 USDC from Arbitrum into a Base vault
curl -X POST https://api.yieldo.xyz/v1/quote \
  -H "Content-Type: application/json" \
  -d '{
    "from_chain_id": 42161,
    "from_token": "0xaf88d065e77c8cC2239327C5EDb3A432268e5831",
    "from_amount": "1000000000",
    "vault_id": "8453:0xbeefe94c8ad530842bfe7d8b397938ffc1cb83b2",
    "user_address": "0xYourAddress"
  }'

# 3. Build the transaction
curl -X POST https://api.yieldo.xyz/v1/quote/build \
  -H "Content-Type: application/json" \
  -d '{
    "from_chain_id": 42161,
    "from_token": "0xaf88d065e77c8cC2239327C5EDb3A432268e5831",
    "from_amount": "1000000000",
    "vault_id": "8453:0xbeefe94c8ad530842bfe7d8b397938ffc1cb83b2",
    "user_address": "0xYourAddress"
  }'
```
